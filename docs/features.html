Features Implemented as of 16-Feb-2015:

User features

Home page features

("Hello, " + first_name + " " + last_name) when user is logged in.

User can sign in by clicking the Sign In button and entering their e-mail address and password.

User can sign out by clicking the Sign Out button.

Your Interests section

Your Interests section appears when user is logged in. Otherwise, it is hidden.

The Your Interest section has a heading: "Your Interests".

Under the heading "Your Interests", a list of issues that the user has indicated an interest appears when user is logged in.

Under the last issue, a text box appears with a "+" button to its right.

If the user types in the name of an issue and clicks the "+" button, the issue becomes an interest of the logged-in user.

All Issues section

The All Issues section has a heading: "All Issues".

Under the heading "All Issues", Home page displays a select object containing all issues in the system.

Selecting an issue displays its internal ID to the right of the select object.

Your Members section

Your Members section appears when user is logged in. Otherwise, it is hidden.

The Your Members section has a heading: "Your Members".

Under the heading "Your Members", the representative and senators of the logged-in user appear, based on their district and state.

The display for each member of congress includes their official picture and full name.

District Map section

Below the Your Members section, a map of congressional districts appears.

When mousing over a district, the district turns purple

Sign In page features

User can specify their e-mail address as clear text.

User can specify their password as obscured text.

User can click the Sign in Button to sign in. If their credentials are correct, then they are taken to the home page. If their credentials are not correct, then the sign-in page is re-displayed with an error message at the top

User can click the Create a New Account link to create a new account.

Create a New Account page features

User can create a new account with the following:
  • E-Mail address
  • Password
  • First Name
  • Last Name
  • Street Address
  • Zip Code

User's district and state are calculated via street address and zip code and stored as constituent data.

User can click the Join DICO button, which creates their account, logs them in and returns to the home page

If the user clicks the Join DICO button and the user specifies an e-mail address that is in use for another user, the Create a New Account page re-appears with an error message at the top:

  • Error : UNIQUE constraint failed: custom_user_authuser.email
